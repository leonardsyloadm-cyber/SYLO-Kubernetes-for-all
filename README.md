To whoever it may concern.
I am the main developer of SYLO.
Sylo was born of an idea regarding automatic kubernete provision for small groups of people who lacked access for AWS,
this is a cheap alternative with many functions and features for those who want to use the amazing technology of Kubernetes
without having to spend expensive amounts on it.

Thus said this is not as good as Amazon or other alternatives but I work hard daily to provide the bext experience for all users,
we are an open source alternative for kubernetes, offering as extra information of how to develop ideas as a consulting company
specialiced in Kubernetes.

We are still learning and I will continue to develop this project.

As a secondary feat this is the main repository for 'KyloDB', a newSQL databse manager who uses memory to operate, based on Redis
Mysql and Oracle, it has both a web administration page and compatibility with Mysql within Dbeaver, this is early into development
but I can guarantee this will come up forth.

- Ivan

Para aquel al que le interese,
Soy el desarrollador principal de SYLO.
Sylo nacio sobre la idea de poder provisionar kubernetes automáticamente para cualquier grupo pequeño de personas que no tuviesen 
acceso a AWS, siendo Sylo una alternativa barata al uso de kubernetes comparado con otras compañias de aprovisionamiento.

Habiendo dicho esto, mi proyecto no es tan bueno como AWS u otras compañías de aprovisionamiento de kubernetes, somos desarrolladores
de código Open Source para kubernetes, ofreciendo otra alternativa para kubernetes, como extra también ofrecemos consultoría para
aquellas empresas o particulares que necesiten, especializandonos en kubernetes.

Todavía estamos aprendiendo y seguiré desarrollando este proyecto.

Como proyecto secundario he creado mi propio SGBD(Sistema Gestor de Base de Datos) llamado KyloDB.

KyloDB es un sistema gestor que opera en memoria y está basado en Redis, Oracle y Mysql, tiene tanto una consola de adminstración web
parecida a PhpMyAdmin tanto compatibilidad con Mysql dentro de DBeaver. Ahora mismo esta en proceso de desarrollo pero puedo garantizar
que este sistema será muy fuerte algún día si no me rindo.
